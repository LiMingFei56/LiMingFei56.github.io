## 数据类
kotlin提供数据类来创建javabean,pojo保存数据的类.提供一些额外的函数.使用关键字
data来创建数据类:

	data class User(val name: String, val age: Int)

编译器会使用主构造函数中的参数,自动的创建一些函数:

	- equals()/hashCode()一对;
	- toString()根据主构造函数;
	- componentN()函数,以属性的声明顺序来生成;
	- copy()方法.

数据类的规则:

	- 主构造方法最少需要一个参数
	- 主构造方法的所有参数必须使用"var"或"val"
	- 数据类不能是抽象的,不能被继承,不能是嵌套的,不能是内部的.

数据类的成员规则:
	- 如果在类中或超类中有显式的实现equals(),hashCode(),toString()那么就不自动生
	  成.
	- 如果超类型有一个componentN()函数,并且这个函数标记为"open"和返回兼容的类型,
	  那么编译器会生成一个新的函数来重写这个函数.如果超类型的componentN()函数没
	  "open"标记或者是final的,那么会报错.
	- 不允许对componentN()函数和copy()函数显示的提供实现.

数据类可以继承其他的类.

在jvm中,如果需要生成无参构造函数,需要主构造函数中所有参数提供默认值.

### 1. copy()函数
有时候需要得到一个对象某些属性不一样的拷贝,这个时候可以使用copy函数.

	fun copy(name: String = this.name, age: Int = this.age) = User(name, age)

	val jack = User(name = "Jack", age = 1)
	val olderJack = jack.copy(age = 2)

### 2. 数据类和解构的声明
数据类生成的componentN()函数一般用于解构的声明:

	val jane = User("Jane", 35) 
	val (name, age) = jane
	println("$name, $age years of age") // prints "Jane, 35 years of age"

### 3. 标准数据类
标准库提供了2元数据类"Pair"和3元数据类"Triple",通常推荐使用自定义的数据类,因为可
以定义名字有意义的属性.
