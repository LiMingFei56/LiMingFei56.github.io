  • + - 基本
    • 一切都是对象,可以在任何变量上调用成员函数和属性
    • + - 基本类型
      • + - 数字
        • 数字16进制,2进制表示可以加划线来增加可读性,编译后会去掉
        • 数字表示支持10进制,16进制,2进制,不支持8进制
        • 数字处理跟Java类似,但是没有包装类型(int,Integer)
        • 数字一般使用JVM内置类型来表示,如果数字允许为null,那么保存的是引用,使用封包
        • 封包时不一定会保留身份信息,并且有性能开销
        • 数字没有隐式转换,必须显示转换
        • 数字支持位操作
      • + - 字符
        • 字符用Char表示,不能直接视为一个数字
        • 字符用'1'表示,可以用\来转义,也可以使用unicode'\uFF00'
        • 字符可以显示转换成数字
      • + - 布尔
        • true, false
        • lazy || && !
      • + - 数组
        • 使用Array来表示
        • 使用get,set来获取内容(重载操作符变成[])
        • arrayOf,arrayOfNulls
        • Array(size,function)
        • 数组是不变的
        • 使用专门的数组类可以节略封包的开销,IntArray,intArrayOf
      • + - 字符串
        • 使用String表示
        • String是不可变的
        • 可以使用[]来访问字符
        • 可以使用for-loop来迭代
        • 原生字符串使用""",不转义,支持字符串模板,使用${'$'}来表示$
        • |是trimMargin()默认前缀
        • "$a"是字符串模板
    • + - 包
      • 源文件开头可以从包声明开始
      • 定义的类和函数都属于该包下面
      • 源文件会默认导入一些包,kotlin,jvm,js
      • 使用import来导入其他的包,可以单个,目录下所有,如果包冲突也可以使用as关键字来指定别名
      • 可以使用import来导入顶级方法和属性
      • 可以使用import来导入对象中的方法和属性
      • 可以使用import来导入枚举常数
    • + - 控制流程
      • + - if
        • 在kotlin中if是一个表达式,有返回值,所以不支持?:运算符
        • 如果是代码块,那么最后一个表达式的值就是返回值
        • 如果用IF作为表示式,必须有其他的分支else
      • + - when
        • 替代switch
        • 可以当表达式,或者语句用.块的话取最后一个表达式的值返回
        • 会顺序匹配值,直到满足一个就返回
        • else作为默认值,如果都不满足就返回else值
        • 如果作为表达式,必须有else,除非编译器识别出满足条件的值
        • 条件可以用,号分隔,实现或
        • 条件可以使用表达式
        • 条件可以使用in !in来表范围
        • 条件可以使用is !is来判断类型
        • 可以不传参数来实现if else if表达式
      • + - loop
        • + - for
          • for可以迭代所有可迭代的对象
          • 对象实现iterator,next,hasNext三个方法并用operator来修饰,就是可迭代的
          • 使用array.indices,array.withIndex来获取下标
        • + - while
          • like java
        • + - do-while
          • like java
        • loop支持break和continue
    • + - 跳转
      • 支持3个表达式,return,break,continue
      • 表达式前可以添加标记,使用跳转时可以指令标记loop@
      • lambda表达式return调用会使包含当前lambda表达式的函数返回,如果不希望这样就需要使用标记
      • kotlin支持隐式标记
      • 也可以使用匿名函数代替lambda,匿名函数return只返回本身
  • + - 类和对象
    • + - 类
      • 使用class关键字创建类
      • 每个类可以有一个主构造器,多个二级构造器
      • 主构造器没有方法体,需要实现init方法来初始化
      • 如果有主构造器,二级构造器需要使用this关键字来调用主构造器
      • 如果构造器参数都有默认值,那么JVM会创建一个无参构造器并使用默认值
      • 创建实例不加new关键字Person()
      • class可以包括构造器,init块,方法,属性,嵌套和内部类,对象声明
    • + - 继承
      • kotlin类都是final类型,需要继承的话要声明为open
      • 使用super调用父类构造器
      • 覆盖方法需要用Open关键字修饰被覆盖方法,使用override修饰覆盖方法
      • 属性也可以用open,override来覆盖属性
      • 如果多继承到了同一个方法,需要使用override来处理,使用super<A>.f()来选择调用的方法
      • 使用abstract来修饰抽象类和方法
      • kotlin中没有静态方法,使用Companion Objects来解决
    • + - 属性
      • var定义可变变量,val定义不可变
      • 属性必须在构造器中初始化
      • 属性可以有get,set方法
      • Backing Fields,只能在get,set方法里用
      • 使用const定义常量,只能是主类型,只能定义在对象的最上面,不能有get set
      • 使用lateinit关键字可以稍后初始化属性,但是不能是主类型,不能是空值
    • + - 接口
      • 接口内可以包含实现方法和抽象方法
      • 接口内可以包含属性,但是属性应该是不变的,或者构造器可以实现的
      • 实现多个有冲突的接口,需要使用super来解决冲突
    • + - 可见性修饰
      • 有四种类型private,protected,internal,public.默认public
      • + - package:不支持protected
        • 默认public
        • 如果是private,那么就只能文件内使用
        • 如果是internal,那么就只能同模块内用
      • + - class和interface
        • private,只在内部可以看到
        • protected,private+包括子类
        • internal,模块内可以看到类的定义的地方
        • public,所有可以看到类的定义的地方
      • 覆盖时可以修改可见性,如果不修改默认继承可见性
      • + - 枸造器
        • public默认
        • 可以修饰为private
        • 可以修饰为internal,只相同模块可见
      • 局部变量,函数不具有可以性
      • + - 模块指的是一组一块编译的KOTLIN的文件
        • an IntelliJ IDEA module
        • a Maven or Gradle projec
        • a set of files compiled with one invocation of the Ant task
    • 扩展
      • 支持扩展方法和扩展属性,可以不使用一些设计模式
      • + - 扩展方法
        • 使用"ClassName"."FunctionName"来创建扩展方法
        • ClassName的对象.functionname来调用扩展方法
        • 扩展方法不会对类添加新的成员,
        • 扩展方法是静态调用的,只与调用该函数的类型有关,不与运行时类型有关
        • 扩展方法如果跟类的成员方法签名一样,那么总是调用类的成员方法
        • 扩展方法的接收者可以为空,可为空时需要使用this来判断是否为空
        • 可以为Companion Object创建扩展方法和属性,使用类名直接调用
        • 可以在类里定义另一个类的扩展,当前类为调用方,扩展的类为被调用方
        • 在类中定义,有多个隐含的接收者,使用this@C.foo来指定是哪一个
        • 在类中定义的扩展,可以被继承和覆盖
      • + - 扩展属性
        • 扩展属性使用val "ClassName"."PropertiesName"来定义
        • 扩展属性不能被初始化
        • 扩展属性不能有set方法,只能有get方法
        • 扩展属性只能用val来定义,不能用var

    • + - 委托属性
      • + - 使用的环境
        • + - 懒加载
          • val lazyValue: String by lazy
          • 默认使用synchronized
          • LazyThreadSafetyMode.PUBLICATION,支持多线程
          • LazyThreadSafetyMode.NONE,单线程
        • + - 观察者属性
          • var name: String by Delegates.observable("<no name>")
        • + - 把属性存储到Map里
          • val name: String by map
      • 声明val/var <property name>: <Type> by <expression>
      • 需要实现getValue,setValue方法
      • 可以在函数,代码块中声明委托属性,不一定要成员
      • val只读属性需要实现一个getValue,有2个参数thisRef,property
      • var除了getValue,还要实现一个setValue,有3个参数thisRef,property,value
      • 可以继承接口,ReadOnlyProperty,ReadWriteProperty
      • 委托属性会生成一个辅助属性来处理如,prop,会生成一个prop$delegate = MyDelegate()
      • prop的get,set方法会调用辅助属性的getValue,setValue
      • thisRef就是外部类的对象,Kproperty就是prop的反射对象
      • 如果by右侧是operator fun provideDelegate方法,那么会使用该方法来创建委托对象
      • 可以使用单独的类和扩展函数实现provideDelegate

      • 总是被花括号括起来
